# Repository Guidelines

## Remote Work Notes
- This environment is usually accessed through SSH. Do not assume local Obsidian notes are available.
- For debugging, setup, deployment, or system administration work, leave a concise work note under `~/work-notes/`.
- Use filename format `YYYY-MM-DD-HHMM-topic.md`.
- Include these sections: Context, Goal, Commands Run, Files Changed, Findings, Resolution, Follow-up, Sensitive Data Policy.
- Do not record secrets, passwords, API keys, private keys, full tokens, or private customer data. Use aliases such as `robot01`, `training-server`, or `internal-registry` for sensitive host or service details.
